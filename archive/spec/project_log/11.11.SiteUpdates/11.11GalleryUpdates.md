## API Routes

### `/api/gallery` - Gallery Management API

#### GET - Fetch Images
```typescript
GET /api/gallery?destination=adult-class
```
- **Query Params**: `destination` (optional, comma-separated)
- **Response**: `{ success: true, data: PictureGalleryItem[] }`
- **Filtering**: Supports multiple destinations
- **Ordering**: By creation date (newest first)

#### POST - Upload Images
```typescript
POST /api/gallery
Content-Type: multipart/form-data
```
- **Body**:
  - `title` (required)
  - `description` (optional)
  - `destinations` (required, JSON array)
  - `file_0`, `file_1`, ... (files)
- **Process**:
  1. Validates inputs
  2. Uploads each file to Sanity assets
  3. Creates pictureGallery documents
- **Response**: `{ success: true, data: [...], message: "..." }`

#### PUT - Update Metadata
```typescript
PUT /api/gallery
Content-Type: application/json
```
- **Body**: `{ id, title, description, destinations }`
- **Process**: Updates document metadata in Sanity
- **Response**: `{ success: true, data: updatedDocument, message: "..." }`

#### DELETE - Remove Image
```typescript
DELETE /api/gallery?id=<imageId>
```
- **Query Params**: `id` (required)
- **Process**:
  1. Fetches document to get asset reference
  2. Deletes document
  3. Attempts to delete asset (if not used elsewhere)
- **Response**: `{ success: true, message: "..." }`

## Technical Implementation

### Architecture Decisions

#### 1. **Destination as Array**
- Stored as `string[]` in Sanity for flexibility
- Allows images to appear in multiple galleries
- Requires Sanity schema update: `type: 'array', of: [{type: 'string'}]`

#### 2. **Backward Compatibility**
- Made `destination` optional in TypeScript: `destination?: string[]`
- Added safety checks throughout code
- Displays "No destination set" badge for old images
- Allows editing to add destinations

#### 3. **Image Presentation**
- Changed from `object-cover` to `object-contain`
- Ensures full image visibility (no cropping)
- White cards with padding for clean presentation
- Title/description below image instead of overlay

#### 4. **Responsive Layout**
```css
/* Mobile */
.carousel-item { width: 100%; }

/* Tablet */
@media (min-width: 768px) {
  .carousel-item { width: calc(50% - 0.5rem); }
}

/* Desktop */
@media (min-width: 1024px) {
  .carousel-item { width: calc(33.333% - 0.667rem); }
}
```

#### 5. **Smart Scrolling**
- Calculates items to scroll based on viewport width
- Desktop: scrolls 3 items at a time
- Tablet: scrolls 2 items at a time
- Mobile: scrolls 1 item at a time

### Helper Functions

#### Gallery Query Builder
```typescript
buildGalleryQuery(destinations?: string[]): string
// Returns: *[_type == "pictureGallery" && ("adult-class" in destination || "kid-class" in destination)]
```

#### Destination Counter
```typescript
countItemsByDestination(items: T[]): Record<string, number>
// Returns: { "adult-class": 5, "kid-class": 3, ... }
```

### IMPORTANT: Manual Step

Update the pictureGallery schema in Sanity Studio:

**File:** `/Users/benjamincorbett/code/cedesigns/studio-coastalcreationsstudio/schemaTypes/pictureGalleryType.ts`

**Change:**
```typescript
// FROM:
defineField({
  name: 'destination',
  title: 'Destination',
  type: 'string',
  validation: (rule) => rule.required(),
})

// TO:
defineField({
  name: 'destination',
  title: 'Destination',
  type: 'array',
  of: [{type: 'string'}],
  validation: (rule) => rule.required().min(1),
})
```

**After updating:**
- Redeploy Sanity Studio
- Existing images without destinations will be handled gracefully
- New images will support multiple destinations
